# ohcha-banner-textsimple

> このスキルの用途:
> テキスト主役のフラットでシンプルなバナー（くすみ単色背景＋白文字＋最小限のあしらい）を
> 固定のトーン＆マナーで制作する、または画像生成 AI / 他エージェント用プロンプトを吐き出すスキル。
> 「シンプルなバナー」「テキストだけのバナー」「textsimple で」「お知らせバナー」
> 「クーポンバナー」「送料バナー」「LINE 訴求バナー」などのリクエストで必ず使用する。
> 日本語文字を正確に出すため HTML/CSS 制作を第一選択とし、
> ChatGPT / Codex / 画像生成 AI 向けプロンプト出力にも対応する。
> 写真主役のナチュラルバナーは ohcha-banner-tomisimple を使うこと。

# テキスト主役シンプルバナー（textsimple）スキル

## このスキルの役割

テキストだけで伝えるフラットバナーの実物サンプル 6 枚を分析して抽出した固定トンマナで、
お知らせ・クーポン・送料・ LINE 訴求・メディア実績などの「情報伝達バナー」を再現的に量産する。
分析元サンプル: ohcha-agent リポジトリ `inputs/samplebanner/textonly/`（スキル作成時点）。

- このスキルが明示的に指定された時点で、トンマナの初回確認（ohcha-tone-manner のルール）は完了扱いとする
- ただし特定クライアントブランドの案件で背景色をブランドカラーへ差し替える場合は、差し替え内容をユーザーに確認する
- サンプル内の実在ロゴ・商標は使用しない。借りるのは「作法」だけ

### 使い分け

| 作りたいもの | スキル |
| --- | --- |
| テキストが主役のフラットバナー（お知らせ・クーポン・送料・実績等） | このスキル |
| 写真が主役のバナー（商品・特集・レシピ・セール・カテゴリ） | ohcha-banner-tomisimple |

## トンマナ定義（固定）

### 世界観キーワード

**静かで清潔 / 実用 / 情報がすっと入る**。
装飾で盛らず、色数と要素を削ることで信頼感を出す。バナーというより「サインデザイン」に近い。

### 配色システム（2 系統。1 バナーにつきどちらか 1 つ）

**系統 A: くすみ単色（中明度）× 白文字**

| 色名 | HEX 目安（サンプル実測の近似値） | 用途例 |
| --- | --- | --- |
| リーフグリーン | `#5FB245` | LINE ・友だち登録系 |
| レンガレッド | `#C1554E` | クーポン・重要告知 |
| くすみオリーブ | `#9AA061` | 送料・配送系 |
| スモークブルー | `#6B8CA8` | 会員・サービス案内（拡張色） |
| くすみマスタード | `#C9A24B` | キャンペーン（拡張色） |

**系統 B: 淡ベージュ・生成り × 濃色文字**

| 要素 | HEX 目安 |
| --- | --- |
| 背景 | `#F6EAE1` / `#D8C6AB` |
| 文字 | `#40301D` / `#262626` |
| 背景の線画アイコン | 背景より 1 段濃い同系色（例: `#E7C3A4`） |

**アクセントは 1 色まで**（例: ピル型バッジの黄 `#F0E24B`）。合計 3 色を超えたら失格。

### タイポグラフィ

- **太ゴシックのみ**が基本（Noto Sans JP の Bold〜Black 相当）。明朝は使わない
- ジャンプ率は高く。主役語 : 修飾語 = 2.5:1 以上
  - 例: 「現在使用できる（小）**クーポン一覧**（特大）」
- 数字は特大、単位・条件は小さく
  - 例: 「合計（小）**600**（特大）円（中）**OFF**（特大）」「**6,400** 円（税込）以上のお支払いで（小）**送料無料**（特大）」
- 注記は最下部に「※」付きで小さく置く（例: 「※卸サイトは除く」）
- 縁取り文字・グラデーション・ドロップシャドウは禁止

### あしらい（使ってよいのは以下だけ）

- **ピル型バッジ**: 条件・チャネル名の列挙（例: 「LINE 友だち登録」「LINE ID 連携」）。角丸大、背景はアクセント色または白
- **単色線画アイコン 1 つ**: チケット・トラックなどの意味が一目で分かるもの。白またはごく淡い同系色、ストローク幅は均一
- **背景の線画アイコンパターン**: 背景色より 1 段濃い同系色で、テーマに沿うアイコンを散らす（主役文字の邪魔をしない濃度に抑える）
- **丸囲みの 1 文字**: 白い正円の枠線＋中に漢字 1 文字（例: 「型」）。象徴的に使う
- 使わないもの: 写真の多用、グラデ、影、フレーム装飾、キラキラ、絵文字

### コピーの作法

- 短く、事実だけ。「現在使用できるクーポン一覧」「最短翌日お届け」
- バナーの主コピーは体言止め可（バナーはコピー扱い。サイト本文の「です・ます」ルールとは別）
- 煽り語・「!」の多用はしない

## サイズと配置の考え方

- サイズは固定しない。**案件ごとの指定（掲載面の要求サイズ）に従う**。指定がなければ用途を確認する
- 配置はバナーのサイズ・比率に合わせて臨機応変に組み替える
  - 横長帯なら要素を左右に振り分けて 1 列に、正方形・縦長なら縦に積む
  - 文字サイズは絶対値でなく比率で考える（例: 横長帯の特大文字はバナー高さの 40〜50% が目安）
- 変えてよいのは配置。変えてはいけないのはトーン（配色システム・タイポ・あしらいのルール）

## レイアウトパターン（4 種）

各パターンの配置は目安。実際のバナーサイズ・比率に合わせて臨機応変に組み替える。

### P1 単色フラット帯型（横長比率に向く）

- 背景: 系統 A の単色
- 左: ピル型バッジ縦積み、または線画アイコン 1 つ
- 右: メインコピー（ジャンプ率高く 1 行）
- 例: 「LINE 友だち登録／LINE ID 連携 → 合計 600 円 OFF クーポンをプレゼント」

### P2 センターロゴ・センターコピー型

- 背景: 系統 B の淡ベージュ
- 中央揃えでロゴまたはシンボル＋コピー 2 行（上段小・下段大）
- 静かなブランド訴求・スタジオ案内向け

### P3 アイコンパターン背景型（正方形・縦長比率に向く）

- 背景: 系統 B の淡色＋同系色の線画アイコンを規則的に散らす
- 中央: 特大コピー 2 行（例: 「メディアで紹介されました」。助詞だけ小さく）
- 実績・お知らせ・受賞報告向け

### P4 ダーク写真 × 白文字型（例外的に写真可）

- 均質な暗トーンの写真（道具・素材の敷き詰め等）に限り背景使用可
- 白文字＋丸囲み 1 文字をシンボルに（例: 「型」＋「千代田型 7 月入荷分販売中」）
- 写真が「テクスチャ」として振る舞う場合のみ。被写体が主役になるなら ohcha-banner-tomisimple へ

## モード A: 制作フロー（HTML/CSS 第一選択）

テキスト主役バナーは画像生成 AI を使わず、**HTML/CSS で組んで正確に出す**のが原則。
日本語の文字化けリスクがゼロになり、修正も一瞬で済む。

1. ヒアリング（不明な項目だけ）: 用途 / サイズ / 文言 / 系統 A か B か / パターン
2. HTML 1 ファイルで実装。フォントは Google Fonts「Noto Sans JP」（weight 700 / 900）
3. playwright-cli で実寸の @2x スクリーンショットを撮って PNG 化
4. 出力先: `reports/banners/`（プロジェクトに別の規約があればそちらを優先）。ファイル名 `{用途}-{YYYYMMDD}.png`。HTML も同じ場所に残す

### P1 実装スニペット（横長帯の骨格。サイズは案件指定に置き換える）

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@700;900&display=swap" rel="stylesheet">
<style>
  /* {W}/{H} は案件指定のサイズに置き換える。文字サイズはバナー高さ基準の比率で決める */
  .banner { width: {W}px; height: {H}px; background: #5FB245; color: #fff;
    display: flex; align-items: center; justify-content: center; gap: 28px;
    font-family: "Noto Sans JP", sans-serif; }
  .badges { display: flex; flex-direction: column; gap: 8px; }
  .badges span { background: #F0E24B; color: #3C3C3C; font-weight: 700;
    font-size: calc({H}px * 0.13); padding: 4px 18px; border-radius: 999px; }
  .copy { font-weight: 900; font-size: calc({H}px * 0.22); display: flex; align-items: baseline; }
  .copy .xl { font-size: calc({H}px * 0.48); letter-spacing: -0.02em; }
  .copy .sm { font-size: calc({H}px * 0.15); }
</style>
<div class="banner">
  <div class="badges"><span>LINE 友だち登録</span><span>LINE ID 連携</span></div>
  <p class="copy"><span class="sm">合計</span><span class="xl">600</span><span class="sm">円</span>
    <span class="xl">OFF</span><span>クーポンをプレゼント</span></p>
</div>
```

## モード B: プロンプト出力

「プロンプトだけちょうだい」と言われたら、以下のどちらかを埋めて出力する。
コピーして貼れる自己完結の形にすること。ChatGPT に常設したい場合は
`chatgpt-prompt.md`（このスキルと同じフォルダ）の携帯版を渡す。

### テンプレ A: Codex / Claude Code / ChatGPT 向け HTML 実装指示（推奨）

```
以下の仕様で {指定サイズ} のバナーを HTML/CSS 1 ファイルで実装してください。
- テイスト: フラットデザイン。装飾なし（グラデ・影・縁取り・絵文字は禁止）
- 背景: 単色 {HEX}（くすみカラー）
- 文字: 白 {または #40301D}、フォントは Google Fonts「Noto Sans JP」700/900
- レイアウト: {P1〜P4 の該当パターンの構成を、このサイズ・比率に合う配置に調整して展開}
- 文言: 「{メインコピー}」を特大、「{修飾語}」をその 1/3 以下のサイズで。
  数字は特大、単位（円・%）は小さく
- あしらい: {ピル型バッジ「{文言}」 / 単色線画アイコン（{モチーフ}）1 つ} のみ
- 注記: 最下部に小さく「※{注記}」
- 完成後 @2x で PNG 書き出し
```

### テンプレ B: 画像生成 AI 向け（やむを得ず画像 AI で作る場合）

```
日本の EC サイト用のフラットなテキストバナー（{サイズ・比率}）を作成してください。
【背景】単色 {HEX}。グラデーションなし、テクスチャなし
【文字】「{メインコピー}」を白の極太ゴシックで特大に。「{修飾語}」は小さく。
日本語文字は一字一句正確に描画すること
【あしらい】{ピル型バッジ / 白い線画アイコン（{モチーフ}）} のみ。他の装飾は一切入れない
【禁止】写真、イラスト、グラデーション、影、縁取り文字、3 色以上の使用、絵文字
```

生成後は文字化け・誤字を必ず目視検品する。文字量が多い場合は必ずテンプレ A のルートに切り替える。

## 品質チェックリスト

- [ ] 色は「背景 1 ＋文字 1 ＋アクセント 1」の 3 色以内か
- [ ] 背景はくすみ・アースカラーで、ビビッド原色になっていないか
- [ ] ジャンプ率 2.5:1 以上、数字特大・単位小さめになっているか
- [ ] あしらいは許可リスト（ピル・線画アイコン・パターン・丸囲み文字）内か
- [ ] グラデ・影・縁取り・絵文字を使っていないか
- [ ] 注記が必要な訴求（割引・送料）に「※」注記を添えたか
- [ ] HTML 制作の場合、実寸 @2x で書き出したか
- [ ] 画像生成した場合、日本語の文字化け・誤字がないか
