# ohcha-lp-design

> このスキルの用途:
> ランディングページ（LP）の企画・構成・ライティング・デザイン・運用改善のスキル。
> LP の新規制作・改善・レビューのあらゆる場面で使用する。
> Web サイトと LP はあり方が根本的に違う：LP は 1 ページ・ 1 目的で CV させる縦長ページであり、
> 回遊させる Web サイト（ohcha-web-design）とは設計思想を分ける。
> 中核は PDL モデル＝ターゲットの心理状態で 3 パターン（P: 問題解決型／D: 欲求充足型／
> L: 潜在需要型）を使い分ける論理展開設計。
> 文章は ohcha-writing、トンマナ・配色は ohcha-tone-manner の決定を最優先し、
> マイクロコピー（ボタン・フォーム・ CTA 文言）と配色（BASE/KEY/SUB/FONT）の
> 共通レイヤーを本スキルに内蔵する。
> 「LP を作って」「LP の構成を考えて」「LP のワイヤーを作って」「CVR を改善して」
> 「広告用のページを作って」などで必ず使用する。
> 参考文献：中尾豊『LP 最強の 3 パターン 制作・運用の教科書／制作ワークブック』（つた書房）、
> 野津瑛司『「最強の一言」Web コピーライティング』、『WEB サイトの配色』を実務向けに再構成。

# ohcha LP 制作スキル

## このスキルの役割

LP は「1 ページ・ 1 目的（CV）」の説得装置。回遊が前提の Web サイトとは別物として設計する。
売れる LP ＝ ①ターゲットの心理状態に合った**論理展開（PDL モデル）**を選び、
②各パートを**要素（コピー・画像・証拠）**で埋め、③公開後に**数値で改善し続ける**もの。
「作れば売れる」は誤り。論理展開を先に固め、デザインは後から乗せる順序を崩さない。

## 優先順位（厳守）

1. **制作手順の全体フロー** → ohcha 作成の **ohcha-ai-site-building** の工程管理に従う
2. **文章の基礎品質** → **ohcha-writing**（完読構造・ AI っぽさ除去）
3. **トンマナ・配色の決定** → **ohcha-tone-manner**（ブランド初回は必ず確認。決定後はトンマナシートに従う）
4. そのうえで本スキルが LP の論理展開・セクション構成・マイクロコピー・運用改善を規定する

## 中核フレーム：PDL モデル（最強の 3 パターン）

ターゲットの検索・購買心理で 3 パターンを使い分ける。**モデル選定を誤ると内容が良くても売れない。**

### 選定フロー
1. すでに問題を自覚し、解決を検索している → **P モデル**
2. 商品・カテゴリを「欲しい」と検索している（緊急性なし） → **D モデル**
3. 必要性にまだ気づいておらず、検索すらしていない → **L モデル**

| | P（Problem 問題解決型） | D（Desire 欲求充足型） | L（Latent 潜在需要型） |
|---|---|---|---|
| 商材例 | 修理・緊急対応・トラブル解決 | 嗜好品・ライフスタイル商材 | 保険・予防系・遺言/相続など |
| 相性のよい広告 | 検索連動型（依頼意図の強い KW） | 検索＋ショッピング＋SNS/ディスプレイ | SNS ・ディスプレイのみ（検索広告はほぼ無効） |
| 入口の心理 | 「早く解決したい」 | 「これ欲しいかも」 | 「自分には関係ない」→ 認知的不協和で覆す |

### モデル別の論理展開

**P モデル**: 共感（こんなお悩みでは？）→ 問題の深掘り → 解決策の提示 → 信用 → 行動喚起

**D モデル（7 部構成)**: ①共感・発見 → ②信用 → ③想像（手に入れた後の生活。ベネフィットは機能的/情緒的/自己表現の 3 種で深掘り、「だからどうなる？」を繰り返す）→ ④比較（自社を表の中央に置き装飾で強調）→ ⑤不安解消 → ⑥買う理由＝言い訳づくり（今すぐ喜べる特典＋期限。家族や友人を巻き込む言い訳ほど強い）→ ⑦行動喚起

**L モデル（9 部構成）**: ①認知的不協和（思い込みと矛盾する事実で「え！？」を作る。事実と思い込みのギャップが大きいほど強い）→ ②必要性の自覚 → ③発信者への信用（肩書き・実績・想い）→ ④解決策＝商品説明（**早く出しすぎない**。必要性の自覚前に出すと雑音）→ ⑤商品への信用 → ⑥比較 → ⑦不安解消（試しやすさ: 無料体験・返金保証・フロント商品）→ ⑧買う理由 → ⑨行動喚起
- 根拠の信用度は「国・行政 > 研究機関・専門家 > 新聞・ TV > 書籍」の順。なければ自社アンケートで代替
- 高額・長期検討商材はゴールを 1 つに絞らず、心理的ハードルの異なる CTA を 2 つ以上（例: 無料相談＋資料請求）用意し KGI/KPI を分解する

## 制作前ワーク（ヒアリング・記入項目）

LP に着手する前に以下を埋める。埋まらない項目は制作前にクライアント/自分に確認する。

- 商材の特長（事実）と特徴（他社比較の優位点）／価格・オファー
- ターゲットの悩み・欲求・誤解・思い込み（L モデルなら「驚かれる事実」も）
- ベネフィット（機能的／情緒的／自己表現）
- 競合との比較軸（価格・機能・実績・サポート）
- 証拠: お客様の声・実績数値・メディア掲載・発信者プロフィール
- 特典・保証・お試し導線（今すぐ喜べるか・期限はあるか）
- 断られた理由（失注理由。表に出ない本音が拾える）
- 検索キーワードと検索ボリューム（キーワードプランナー・ラッコキーワード等）
- 最終的に取ってほしい行動（CTA。複数の心理的ハードルで用意）

## セクション別ルール

- **ファーストビュー**: 3 秒で「自分に関係ある」と判断される。必須要素＝ロゴ・キャッチコピー・ターゲットコピー・イメージ・行動喚起バナー。重要情報は左（視線の始点）。人物写真は右配置・視線をコピー側（左向き）に。スマホは PC と同じ情報量を詰めない
- **キャッチコピー**: 冒頭 13〜15 文字に最重要ワード。L モデルは 7 つの型（ご存知ですか/実は/話題性/お得感/意外性/禁止（カリギュラ）/試されている）を使う
- **本文**: 60〜100 文字目安で改行（集中力 8 秒前提）。見出し→画像→本文を 1 セットにして流し読みでも伝わる構成に
- **証拠パート**: 実績メダル系の装飾は最低 3 つ並べる（1 つは弱く 2 つは中途半端）。メリットだけでなくデメリットも開示した方が信用され CVR が上がる
- **CTA ・ボタン**: 名詞止めは NG、**動詞＋得られるもの**で書く（「お問い合わせ」→「無料で相談する」）。「送信」「確認」「詳しくはこちら」などシステム目線の文言も NG。**ハードルは商材で調整**する：低価格・即決型は「今すぐ購入する」の後押しが効き、高額・検討型は一段低いステップ（「まずは無料で体験してみる」「料金を確認する」）に下げる。ボタン直下にクリックトリガー（「いつでもキャンセルできます」「クレジットカード不要」等の不安払拭）。ボタン文言と遷移先見出しを一致させる（メッセージマッチ）。広告文と FV の一致も必須。縦長 LP ではメイン CTA をスクロール追従（フローティング）にして常時タップ可能にする
- **カートまわり 8 要素**: 商品画像（大小）／商品名／価格／特徴・優位性／比較表／送料／数量選択／カートボタン。離脱直前の最終アピールの場なので手を抜かない
- **フォーム（EFO）**: 項目最小限・連絡先は早めに確保・リアルタイムバリデーション。項目が多い場合は**ステップ分割**する（1 画面 1 機能。例: ①アカウント情報 → ②基本情報 → ③決済情報）。必須項目は「※」でなく「**必須**」と書き項目名の前に置く。電話番号・生年月日など個人情報には**必要な理由を明記**する（理由なき要求は離脱の大きな原因）。エラー文言は「状態」でなく「次の行動」を示す
- **付随情報の置き方**: FAQ ・詳細スペック等は別ページに逃さず、アコーディオン（開閉 UI）で同一ページ内に置く（LP から離脱経路を作らない）
- **フォローアップ**: カゴ落ちは平均 7 割、追撃メール/LINE で約 4 割戻せる。件名は短く、本文で行動を明示

## デザイン数値基準

- **1 カラム構成**（サイドバーなし）。視線を集中させ離脱経路を作らない
- 色は基本 **2 色**（メイン＋アクセント）。メインはコーポレートカラーか商材の連想色、アクセントは補色。色数が増えるほど野暮ったくなる。詳細配色はトンマナシート（BASE/KEY/SUB/FONT ・ WCAG 4.5:1）に従う
- フォントサイズ **16〜18px**（14.6px 未満は不可）
- 説明文は 400 字以内。3 行超えたら箇条書き。漢字 4 文字以上の連続はひらがなに開く
- 画像文字（キャッチ・見出し＝インパクト重視）と HTML テキスト（本文＝修正容易・ SEO）を使い分ける。全部画像文字にしない
- 画像は JPEG 基本。表示速度は PageSpeed Insights 90 点以上を目標（1→3 秒で直帰率 +32%、1→5 秒で +90%）
- パート（論理展開の要素）ごとに区切って実装し、改善時に差し替えやすくする

## 公開後の運用・改善

- CPA ＝ 広告費 ÷ CV 数。目標 CPA ＝ 上限 CPA（粗利）− 確保したい利益。LTV（客単価×リピート）を踏まえ、CPA を下げすぎてアクセスを絞る本末転倒を避ける
- A/B テストは 1 度に 1 要素だけ。1 回の結果で諦めない（戻せば損失ゼロ）
- 改善の優先順は「モデル選定ミス > FV > CTA ・オファー > 証拠 > 細部の文言」
- 注文完了画面でのクロスセル（一貫性の原理＋期限）も検討する

## アンチパターン

- ターゲットの心理状態と合わないモデル選定（L モデル商材に検索広告、など）
- L モデルで商品説明を早く出しすぎる／驚きだけ作って根拠を示さない（怪しさが残る）
- 「次回使えるポイント」などリピート前提の特典（初回訴求には効かない。今すぐ喜べる特典にする）
- CTA が 1 種類だけで心理的ハードルが高い（高額商材は特に）
- PC デザインをスマホにそのまま流用して FV が収まらない
- 全文字を画像化して装飾過多・修正不能にする
- 共感パートなのに能天気なトーンなど、パートの意図とデザインの不一致
- メリットのみの一方的訴求（人は基本信じない前提で証拠とデメリット開示）
- 「作れば売れる」と考え、商品価値の検証をせず形だけ整える

## 品質ゲート（公開前の合格基準）

1. PDL モデルの選定根拠（ターゲットの心理状態）を説明できるか
2. FV だけで「誰の・何の・どうなれる」ページか伝わり、広告文と一致しているか
3. ボタン文言は行動結果を明示し、クリックトリガーで不安を先回りしているか（NG 文言: 送信／確認／詳しくはこちら）
4. 証拠（実績・声・比較）とデメリット開示があり、一方的な訴求になっていないか
5. フォームは必須表記・理由明記・ステップ分割ができているか
6. スマホ実機で FV ・タップ領域・表示速度を確認したか

## 他スキルとの連携

- Web サイト（回遊型）は **ohcha-web-design**、上流の 3C/ターゲット設計は **ohcha-feature-page-creation**（Phase 1）と併用
- 施策全体でのこの LP の位置づけ（どのフェーズの受け皿か）は **ohcha-digital-marketing-teiseki** で確認
- 公開前チェックは **site-review**、ROI 試算は **roi-simulator**
