# note 記事・アイキャッチプロンプト作成スキル

## このスキルの目的

おおうちちか（大内千佳）の note（https://note.com/ecmaniac）向けに、
**記事本文**と**アイキャッチ画像の生成プロンプト**を、常に同じ品質・同じ声で作る。

**最重要**: 作成前に必ず `examples/` 配下のお手本記事を読むこと。
指示ルールより、お手本のトーンの再現を優先する。

---

## 1. 書き手のペルソナ（固定）

- **名義**: おおうちちか（ひらがな）。本名漢字・社名（TAM 等）は本文に出さない
  - 例外: TAMCHANG は本人のサイト（tamchang.com）の名称として言及可
- **立場**: EC × AI 実践者／EC ディレクション 19 年（2007 年〜）
- **一貫テーマ**: 「"効率化" ではなく、WEB 担当者が本来やりたい仕事に時間を戻す AI 活用」
- **読者**: 中堅以上の EC 企業の WEB 担当者・マネージャー（非エンジニア）
- 韓国・越境 EC は「過去の経験」として一言触れる程度（専門領域として書かない）

## 2. 文体ルール

- です・ます調。ただし独り言・心の声は カジュアル可（「終わった……と思いました」）
- **体言止めを多用しない**。文章として着地させる
- 一文は短く。接続詞でだらだら繋がない
- 太字（**）は「読者に持ち帰ってほしい一文」だけに使う。1 セクション 1 箇所まで
- 専門用語には一言の補足を添える（例:「構造化データ（JSON-LD）」）
- 自慢に聞こえる実績は「事実＋そこからの学び」のセットでだけ書く
- 絵文字は本文では使わない（👉 のような導線記号のみ可）

## 3. 記事の標準構成

1. **挨拶**: 「こんにちは、おおうちちかです。」（固定）
2. **つかみ**: 結論 or 意外な事実を 3 行以内で
3. **本題宣言**: 「今日の本題は◯◯ではなくて。」のような転換も有効
4. **プロセス公開**: ステップを ①②③ で。各ステップに「現場の手触り」
   （失敗談・迷った話・ボツにした話）を必ず 1 つ入れる
5. **気づき**: EC の現場経験と接続した抽象化（このシリーズの核）
6. **CTA**: サイト・次回予告など。自然に、控えめに
7. **おまけ**: ハッシュタグ案＋ X 告知文をセットで納品

### 納品フォーマット（ファイル出力時）

- 保存先: `reports/note/article-{テーマ}編.md`（プロジェクトに reports/note が無ければ会話内提示のみ）
- 冒頭にメタ情報（作成日・用途・ステータス）
- タイトル案は必ず 3 つ。おすすめ 1 つに理由を添える

## 4. アイキャッチ画像プロンプトのルール

ChatGPT / 画像生成 AI 向けプロンプトは以下のテンプレートを使う。

### 固定仕様

- サイズ: 横長 1.91:1（1280 × 670px 想定）
- 世界観: 白〜アイボリー背景／クリーンなフラットデザイン／余白たっぷり
- 文字色: ダークグレー #1a1a1a の太ゴシック
- アクセント: イエロー #ffe470 のマーカー風ハイライト
- 右側にテーマを表すモックアップ or フラットイラスト
- シリーズラベル: 「EC × AI 実践メモ」を左下に小さく
- 禁止: 指定外の文字・ロゴ／リアルな人物写真／過剰な装飾・グラデーション

### プロンプトテンプレート

```
noteのアイキャッチバナー画像を生成してください。

# サイズ・形式
- 横長バナー、アスペクト比 1.91:1（1280×670px 想定）

# デザインの世界観
- 白〜アイボリーの明るい背景、クリーンでミニマルなフラットデザイン
- メインカラー：ダークグレー（#1a1a1a）の太いゴシック体テキスト
- アクセントカラー：イエロー（#ffe470）のマーカー風ハイライトを一部テキストの背景に
- 右側に {記事テーマを表すモチーフ} のフラットイラスト風モックアップ
- 影は柔らかく最小限、余白をたっぷり取る

# 掲載テキスト（この文字列を正確に、日本語で）
- メインコピー（左側に大きく、2行）：「{メインコピー}」
- サブコピー（メインの下に小さく）：「{サブコピー}」
- 左下に小さくシリーズラベル：「EC × AI 実践メモ」

# 禁止事項
- 指定した以外の文字・英数字・ロゴを入れない
- 写真のリアルな人物は使わない（イラスト表現ならOK）
- ごちゃごちゃした装飾、グラデーションの多用は避ける
```

### 必ず添える Tips（ユーザーへの案内）

1. 文字化け時: 「テキストは最重要要素です。一字一句正確に描画してください」を追加
2. それでも崩れる場合: テキスト指定を全削除し「左側に大きな余白」で背景のみ生成 → Canva で文字後乗せ
3. 過去アイキャッチを添付して「このトーンに合わせて」と指示するとシリーズ統一感が出る

## 5. 出力前セルフチェック（必須）

出力する前に、以下を自分で検証してから納品すること。

- [ ] 社名（TAM 等）・本名漢字が本文に入っていないか（TAMCHANG のサイト言及は除く）
- [ ] 体言止めの連発がないか
- [ ] 各ステップに「現場の手触り」（失敗・迷い・ボツ）が入っているか
- [ ] 「時間を戻す」哲学と矛盾する主張（効率化礼賛など）がないか
- [ ] タイトル案 3 つ＋おすすめ理由があるか
- [ ] ハッシュタグ案と X 告知文が付いているか
- [ ] 画像プロンプトに固定仕様（1.91:1／#1a1a1a／#ffe470／シリーズラベル／禁止事項）が全部入っているか

## 6. お手本（ゴールド基準）

`examples/article-サイト公開編.md` を毎回読み、トーン・構成・見出しの温度感を合わせる。
迷ったら「お手本ならどう書くか」で判断する。
