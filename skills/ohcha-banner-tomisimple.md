# ohcha-banner-tomisimple

> このスキルの用途:
> 富澤商店系のナチュラル EC バナー（実写写真主役・くすみアクセント・上品なシズル感）を
> 固定のトーン＆マナーで制作する、または画像生成 AI / 他エージェント用プロンプトを吐き出すスキル。
> 「富澤商店ぽいバナー」「富澤風で」「tomisimple で」「ナチュラルな EC バナー」
> 「食品系のバナーを作って」「特集バナー」「レシピバナー」「カテゴリバナーを統一感を持って作りたい」
> などのリクエストで必ず使用する。
> 出力は 2 モード。A) バナーそのものを制作（写真生成＋ HTML/CSS 文字合成が基本）、
> B) ChatGPT / Codex / 画像生成 AI にそのまま貼れるプロンプトを出力。
> テキスト主役のフラットバナーは ohcha-banner-textsimple を使うこと。

# 富澤商店風ナチュラルバナー（tomisimple）スキル

## このスキルの役割

富澤商店（製菓・製パン材料 EC）のバナー実物 26 枚（プロモバナー 18 枚＋カテゴリバナー 8 枚）を
分析して抽出した固定トンマナで、同品質のバナーを再現的に量産する。
分析元サンプル: ohcha-agent リポジトリ `inputs/samplebanner/tomizawa/`（スキル作成時点）。

- このスキルが明示的に指定された時点で、トンマナの初回確認（ohcha-tone-manner のルール）は完了扱いとする
- ただし特定クライアントブランドの案件でアクセント色をブランドカラーへ差し替える場合は、差し替え内容をユーザーに確認する
- 富澤商店の実ロゴ・商標・商品パッケージそのものは使用しない。借りるのは「作法」だけ

### 使い分け

| 作りたいもの | スキル |
| --- | --- |
| 写真が主役のバナー（商品・特集・レシピ・セール・カテゴリ） | このスキル |
| テキストが主役のフラットバナー（お知らせ・クーポン・送料等） | ohcha-banner-textsimple |

## トンマナ定義（固定）

### 世界観キーワード

**上質な日常 / 素材の良さ / 手仕事 / 清潔感 / 静かなシズル**。
セールバナーでも「量販店の安売り感」を絶対に出さない。値引きすら上品に伝えるのがこのトンマナの核。

### サイズと配置の考え方

- サイズは固定しない。**案件ごとの指定（掲載面の要求サイズ）に従う**。指定がなければ用途を確認する
- レイアウトパターン内の配置（右上・左下など）はすべて「目安」。**バナーのサイズ・比率・写真の余白に合わせて臨機応変に調整する**
  - 横長なら要素を左右に振り分ける、縦長・正方形なら縦に積む、など比率に応じて組み替えてよい
- 変えてよいのは配置。変えてはいけないのはトーン（写真規定・配色・タイポ・あしらいのルール）

### 写真規定（最重要）

- **実写のみ**。イラスト・ 3D ・ AI 感の強い過剰演出は不可
- 光は自然光。やわらかい影。白飛び・黒潰れさせない
- 背景素材は木目・大理石・リネン・無地の漆喰壁・すりガラスなどのナチュラル素材
- 構図は「シズル感のある寄り」「俯瞰・斜俯瞰のスタイリング」のどちらか
- 彩度は上げすぎない。色温度はニュートラル〜やや暖色
- 暗めのシックな写真（こげ茶・黒背景）も可。その場合は文字を白にする

### 配色（写真がベース。文字・あしらいは以下から最大 2 色）

| 役割 | 色名 | HEX 目安（サンプル実測の近似値） |
| --- | --- | --- |
| 文字（明るい写真上） | こげ茶〜黒 | `#3C3C3C` / `#4A3428` |
| 文字（暗い写真上） | 白 | `#FFFFFF` |
| 割引バッジ | くすみレッド | `#A6534E` |
| 割引バッジ | テラコッタ | `#B96A3D` |
| 帯・バッジ | くすみオレンジ | `#E8862F` |
| 記念・上質バッジ | くすみゴールド | `#A79A5F` |
| セール タイポ | ネイビー | `#4C5C8A` |
| セール タイポ | 朱赤 | `#E85B4B` |
| フラット背景 | ライトグレー | `#E9E9E7` |
| フラット背景 | スカイブルー | `#62C1DC` |
| フラット背景 | ライトブルー | `#D2E9F5` |
| 乳白パネル | 半透明白 | `rgba(255,250,245,0.78)` |

ビビッドな原色・蛍光色・金ピカは禁止。

### タイポグラフィ

- 基本は**太ゴシック**（Noto Sans JP / Zen Kaku Gothic New の Bold〜Black 相当）
- レシピ特集・季節・上質訴求は**明朝**（Noto Serif JP / しっぽり明朝相当）
- 英語アクセントを積極的に併用する: セリフ体大文字（トラッキング広め）または筆記体
  - 例: 「Peach ＋ 桃のレシピ 50 選」「CHOCO MINT ＋ チョコミントレシピ 10 選」「TEA & SYRUP」
- ジャンプ率は高く。主役語 : 修飾語 = 3:1 以上
- 数字は特大、「%」「円」「税込」「MAX」などの単位・補足は小さく添える
  - 例: 「**1,428**円（円は小さく）＋ 税込（さらに小さく）」「MAX（小）**20%OFF**（特大）」
- 和テイストは縦書き明朝も使う（例: 「自由に楽しむ夏茶」）
- 縁取り文字・グラデーション文字・濃いドロップシャドウは禁止（可読性のための薄い影のみ可）

### あしらい

- **円形バッジ**: 割引率・特典。右上または左中段。くすみアクセント 1 色＋白文字。円は正円
- **角丸帯**: キャンペーン条件・期間。上部に置く（例: 「新規会員登録後、初めてのお買い物限定」）
- **商品パッケージの切り抜き**: 右下に置く（価格訴求バナーの定石）
- 手書き風は最小限: 「＼ ＼」スラッシュ、細いリボン型の線画、白フチの円
- 印章風の赤い落款マーク（和テイストのみ）
- 使わないもの: 爆発フキダシ、ギザギザバッジ、斜めテープの多用、キラキラ、絵文字

### コピーの作法

- 短く、事実ベース。「桃のレシピ 50 選」「7 月入荷分販売中」「最大 20%OFF」
- 煽り語（今すぐ急げ・売り切れ御免の連呼）は使わない。「!」は 1 バナーに 1 つまで
- 期日は具体的に書く: 「7.31（金）23:59 まで」「なくなり次第終了」

## レイアウトパターン（6 種）

各パターンの配置指定は目安。実際のバナーサイズ・比率・写真の余白に合わせて臨機応変に組み替える。

### P1 商品プロモ型（価格・割引訴求）

- 写真: 商品の完成品（焼き上がったパン等）をフルブリード
- 右上: 円形割引バッジ（くすみレッド or テラコッタ）「今なら 10%OFF!」
- 左下: 商品名（太ゴシック大）→ 価格（特大数字＋小さい円・税込）
- 右下: 商品パッケージ切り抜き
- 任意: 上部に角丸帯で条件、最下部に小さい注記

### P2 特集・レシピ型

- 写真: 料理・スイーツの寄り。フルブリードまたは中央主役
- 下部: タイトル。「英語セリフ特大＋日本語小」または「日本語明朝特大」
- タイトルの上に 1 行のリード文（小さめ）「さわやかなレモンアイテムで作る」
- バッジ・価格は置かない。写真とタイトルだけで成立させる

### P3 和・季節型

- 写真: 季節感のあるスタイリング（影・光を活かす）
- 右側: 縦書き明朝（漢字 1〜2 文字は特大に）
- アクセント: 筆記体英語、印章風の赤マーク
- 例: 「自由に楽しむ夏茶 ＋ TEA & SYRUP」「台湾スイーツ ＋ Taiwan Sweets」

### P4 フラットセール型

- 背景: ライトグレー / スカイブルー / ライトブルーのフラット単色（または淡色グリッド）
- 商品切り抜き写真を複数配置（下部 or グリッド状）
- タイポ: 2 色まで（ネイビー×朱赤、白のみ等）で「とっておき SALE」「TIME SALE」を特大に
- 右上: 円形バッジ「7 月限定」
- 期日を「7/14（火）昼 11:59 まで」の形式で明記

### P5 乳白パネル型

- 写真: 商品をグリッド状・敷き詰めで撮ったものをフルブリード
- 中央: 乳白半透明パネル（角は直角）を置き、こげ茶文字で「チョコレート CHOCOLATE SALE」
- 文字は 日本語ゴシック＋英語セリフ の 2 段組

### P6 カテゴリ統一型（→ 詳細は「カテゴリバナー統一企画モード」）

- 写真フルブリード＋左下に白太ゴシック 1 語のみ。他の要素は一切置かない

## モード A: バナー制作フロー

### 1. ヒアリング（最小限）

不明な項目だけ確認する: 用途 / サイズ / 文言（商品名・価格・割引・期日） / 素材写真の有無 / パターン希望。

### 2. 写真の用意

優先順位: **ユーザー支給の実写 > 画像生成**。
生成する場合は gemini-image（Nano Banana 2）または gpt-image スキルで、
**文字を一切入れない写真** を上記「写真規定」に沿って生成する（プロンプトはモード B のテンプレ A を使用）。

### 3. 文字合成（基本ルート: HTML/CSS）

日本語を正確に出すため、写真を背景にした HTML/CSS で文字・バッジを組み、
playwright-cli で指定サイズの @2x スクリーンショットを撮る。
フォントは Google Fonts の Noto Sans JP / Noto Serif JP を読み込む。

### 4. 一発生成ルート（補助）

文字ごと画像生成する場合は Nano Banana 2 / gpt-image に日本語文字を含めて指示できるが、
**生成後に文字化け・誤字を必ず目視検品**する。文字量が多い場合はこのルートを選ばない。

### 5. 出力先

`reports/banners/`（プロジェクトに別の規約があればそちらを優先）。
ファイル名は `{用途}-{YYYYMMDD}.png`。HTML も同じ場所に残す。

## モード B: プロンプト出力

「プロンプトだけちょうだい」と言われたら、以下のテンプレを埋めて出力する。
コピーして ChatGPT / Codex / 画像生成 AI にそのまま貼れる自己完結の形にすること。
併せて `chatgpt-prompt.md`（このスキルと同じフォルダ）にトンマナ定義の携帯版があるので、
「ChatGPT に常設したい」と言われたらその内容を渡す。

### テンプレ A: 背景写真のみ生成（英語 / 文字なし）

```
Professional food product photography for a Japanese artisan baking-ingredients online store.
Subject: {被写体の説明}
Composition: {close-up sizzle shot / overhead flat-lay}, subject positioned {center / upper two-thirds},
leaving clear negative space at {bottom-left / bottom / right} for text overlay.
Surface and props: {light wood table / white marble / linen cloth / rustic dark wood}, minimal styling.
Lighting: soft natural window light, gentle shadows, neutral-to-warm color temperature.
Mood: quiet, premium, handcrafted, clean. Muted natural colors, NOT oversaturated.
Strictly no text, no letters, no logos, no watermark, no illustration, no people unless specified.
Aspect ratio {指定比率}, high resolution.
```

### テンプレ B: 全部入り生成（日本語 / Nano Banana 2 ・ GPT Image 向け）

```
日本の製菓材料ECのバナー（{指定サイズ・比率}）を作成してください。
【パターン】{P1〜P5 のいずれかの説明を、この比率に合う配置に調整して展開}
【写真】{被写体}。自然光、やわらかい影、{木目/大理石/リネン}の背景。彩度控えめで上質なナチュラルテイスト。
【文字要素】
- {位置}: 「{文言}」（{太ゴシック/明朝}、{白/こげ茶}、特大）
- {位置}: 円形バッジ（{#A6534E などの HEX}）に白文字で「{割引文言}」
- 最下部: 「{注記}」（小さく）
【文字の規定】日本語文字は正確に描画すること。ジャンプ率を高く（主役語は修飾語の3倍以上）。
数字は特大、「円」「%」「税込」は小さく添える。
【禁止】縁取り文字、グラデーション、濃いドロップシャドウ、ビビッド原色、爆発フキダシ、絵文字、
量販店的な安売り演出。全体を上品に保つこと。
```

### テンプレ C: Codex / Claude Code 向け HTML 実装指示

```
以下の仕様で {指定サイズ} のバナーを HTML/CSS 1 ファイルで実装してください。
- 背景: {写真ファイルパス} を object-fit: cover でフルブリード
- フォント: Google Fonts「Noto Sans JP」(700/900)、{明朝を使う場合は「Noto Serif JP」}
- 文字要素: {P1〜P6 の該当パターンの構成要素と文言・色 HEX を、このサイズ・比率に合う配置に調整して列挙}
- 禁止: text-shadow の濃用、グラデーション、縁取り、3 色以上の使用
- 完成後 @2x で PNG 書き出し
```

## カテゴリバナー統一企画モード

「カテゴリバナーを統一して作りたい」「シリーズで揃えたい」と言われたら、
1 枚ずつ作らず**先にシリーズ全体を設計**してから量産する。

### フォーマット規定（全枚数で完全固定）

- サイズは案件指定（全枚数で同一サイズ）。実写フルブリード。文字を置く側が写真の「余白面」になるよう選ぶ
- 文字は **1 語のみ**。白・太ゴシック（Noto Sans JP Black 相当）。位置は左下が基本だが、シリーズ設計時に決めた位置・サイズを全枚数で固定する
- 日本語: 文字サイズはキャンバス短辺の約 9〜11%。端からの余白は短辺の 7〜8% が目安
- 英語: 大文字のみ、トラッキング +0.15em（例: NEW / TREND）
- 句読点・記号・バッジ・ロゴは置かない。可読性が足りない場合のみ写真下部にごく薄い暗幕グラデ（black 0→18%）を許可

### 進め方

1. シリーズ一覧をテーブルで設計してユーザーに確認する

   | # | カテゴリ名（表記そのまま） | 被写体案 | 写真トーン |
   | --- | --- | --- | --- |
   | 1 | 米粉 | 米粉スコーンと花柄皿 | 暖色・木テーブル |
   | 2 | 抹茶 | 抹茶プリン | 暖色・シック |

2. 写真トーンは「全体で統一」か「各カテゴリの世界観優先」かを確認（サンプルは各カテゴリ優先で、明るさだけ揃えている）
3. 全枚数を同じ文字規定で量産し、グリッドに並べたプレビューを作って統一感を最終確認する

## 品質チェックリスト

- [ ] 写真は実写クオリティで、自然光・ナチュラル素材・彩度控えめか
- [ ] 文字・あしらいの色は 2 色以内で、指定パレットのくすみカラーか
- [ ] ジャンプ率 3:1 以上、数字特大・単位小さめになっているか
- [ ] 縁取り・グラデ・濃い影・ビビッド原色・絵文字を使っていないか
- [ ] 日本語の文字化け・誤字がないか（画像生成した場合は特に）
- [ ] セールでも上品か（安売り感が出ていないか）
- [ ] 富澤商店の実ロゴ・商標を写し込んでいないか
- [ ] カテゴリ統一モードの場合、全枚数で文字サイズ・位置が完全に同一か
