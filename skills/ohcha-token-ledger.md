# ohcha-token-ledger

> このスキルの用途:
> AI 利用コストの経理スキル（経理 AI の役割）。
> Claude 等の利用トークン数から API 単価で費用を計算し、円換算して Excel 帳簿に記帳する。
> 案件別の AI 原価管理・月次集計・見積もりへの原価反映に使う。
> 「トークン代を計算して」「AI のコストを帳簿につけて」「今月の利用料をまとめて」
> 「この案件の AI 原価は？」「トークン帳簿を作って」などで必ず使用する。
> 単価と為替は変動するため、本スキルは計算手順と帳簿フォーマットを規定し、
> レートは実行のたびに最新を確認する。Excel 出力は xlsx スキルを併用する。

# ohcha トークン経理・帳簿スキル

## このスキルの役割

AI 活用の事業（HP 制作・コンテンツ制作）で、**案件ごとの AI 原価**を円で見える化する。
どんぶり勘定を防ぎ、見積もり・値付け・粗利判断（ohcha-small-business / ohcha-b2b-marketing）の土台データを作る。

## 大原則

1. **単価・為替をハードコードしない**: モデル単価も USD/JPY レートも変動する。**記帳のたびに最新値を確認**し、使った値を帳簿の行に必ず記録する（後から検証できる状態にする）
2. **API 換算の参考原価**: 定額プラン（サブスクリプション）で使った分は実費ゼロでも、**「API で同じ量を使ったらいくらか」を参考原価として記帳**する。これが外注時・スケール時の原価見積もりになる（実費と参考原価は列を分けて区別する）
3. **円で読む**: 最終判断に使う列は円（¥・整数）。USD は計算根拠として併記する
4. **案件に紐づける**: 必ず案件（プロジェクト）タグを付ける。案件不明の利用は「internal」として記録する

## 計算手順

### Step 1: トークン数の取得
- Claude Code: セッションの利用実績（`/cost` の表示、完了報告のトークン数、タスク出力の usage 等）から入力・出力トークンを拾う
- API 利用: レスポンスの usage（input_tokens / output_tokens / cache_creation / cache_read）を記録する
- 取れない場合は概算し、帳簿の備考に「概算」と明記する

### Step 2: 最新単価の確認
- Anthropic 公式の料金ページ（docs.claude.com の Pricing）で、使ったモデルの **100 万トークンあたり USD 単価**（入力・出力・キャッシュ別）を確認する。claude-api スキルが使える環境ではそれを参照してよい
- 確認した単価と確認日を帳簿に記録する（単価改定に気づける）

### Step 3: 費用計算

```
USD 費用 = (入力トークン ÷ 1,000,000 × 入力単価)
         + (出力トークン ÷ 1,000,000 × 出力単価)
         + (キャッシュ書込 ÷ 1,000,000 × キャッシュ書込単価)
         + (キャッシュ読取 ÷ 1,000,000 × キャッシュ読取単価)
円換算  = USD 費用 × その日の USD/JPY レート（レートも要確認・要記録）
```

## 帳簿フォーマット（Excel ・ xlsx スキルで出力）

出力先: `reports/token-ledger/AI利用帳簿_{YYYY}.xlsx`

**明細シート（1 行＝1 セッション or 1 作業）**

| 列 | 内容 |
|---|---|
| 日付 | 利用日 |
| 案件 | プロジェクト名（不明は internal） |
| 作業内容 | 例: LP 実装／記事執筆／画像生成 |
| モデル | 例: claude-sonnet-5 |
| 入力トークン／出力トークン | キャッシュ分は備考へ |
| 単価（USD/1M ・入力／出力） | 確認した最新値 |
| USD 費用 | 計算式セル |
| レート（USD/JPY） | 使用レートと確認日 |
| **費用（¥）** | 円・整数。表示形式は ¥#,##0 |
| 区分 | 実費（API）／参考原価（定額プラン内） |
| 備考 | 概算・キャッシュ・画像生成 API 等の特記 |

**集計シート**
- 月次集計: 月×区分（実費/参考原価）の円合計
- 案件別集計: 案件×月の円合計 → **案件の粗利計算・次回見積もりの原価根拠**に使う

## 運用ルール

- 記帳のタイミングは「案件の節目（納品時）」＋「月末」の 2 回を最低線とする（毎セッション記帳は続かない）
- 画像生成（gemini-image / gpt-image）など**トークン建てでない API は「回数×単価」**で同じ帳簿に記帳する（モデル列に API 名）
- 年をまたいだらファイルを分ける。フォーマットは変えない
- 帳簿から得た「作業種別ごとの AI 原価の相場」は、見積もりテンプレの原価項目として再利用する

## アンチパターン

- 古い単価・レートを使い回す（改定に気づかず原価を誤る）
- 定額プランだからと記帳しない（スケール時・外注時の原価が見えなくなる）
- 案件タグなしで合計だけ記録する（どの仕事が儲かっているか分からない）
- USD のまま帳簿を読む（円の意思決定に直結しない）

## 品質ゲート（記帳完了の合格基準）

1. 単価・為替レートに「確認日」が記録されているか
2. 円の列が ¥ 表示・整数で、実費/参考原価の区分があるか
3. すべての行に案件タグが付いているか
4. 集計シートが月次・案件別の両方で合計されているか

## 他スキルとの連携

- Excel の作成・追記: **xlsx** スキル（新規作成・セル編集・シート追加）
- 原価 → 値付け・粗利判断: **ohcha-small-business**（価格設計）／ **ohcha-b2b-marketing**（受注採算）
- クライアント向けの投資対効果説明: **roi-simulator**
- 最新のモデル単価情報: **claude-api** スキル（利用可能な環境のみ）
