# ohcha-tone-manner

> このスキルの用途:
> 案件・ブランド単位のトーン＆マナー（トンマナ）を決定・維持するスキル。
> 提案書などの資料・ Web サイト・ LP ・ SNS/LINE のバナーなど、
> あるブランドのクリエイティブを「初めて」作るときは、必ずこのスキルを使い、
> テイスト・配色案を提示してユーザー（ohcha）に確認・すり合わせしてから制作に入る。
> 一度決定したら同ブランド内は共通トンマナで統一し、トンマナシートとして保存する。
> 1 プロジェクト内に複数ブランドがある場合、どのブランドのトンマナか迷ったら必ず確認する。
> 例外：TAM を紹介する資料・ TAM/TAMCHANG 名義の資料や LP のみ、
> 従来のスライドトンマナ（tam-proposal-deck）を適用する。
> 「トンマナを決めたい」「配色を考えて」「ブランドの世界観を作りたい」
> 「バナーを作って」「新しいクライアントのデザインを始める」などで必ず使用する。
> 参考文献：ingectar-e『トンマナの基本』（ソシム, 2026）、『WEB サイトの配色』の知見を実務向けに再構成。

# ohcha トーン＆マナー決定スキル

## このスキルの役割

ブランドごとに「一貫性 × 時短」を両立するデザインルール（トンマナ）を、ohcha さんと**対話で決定**し、
以降のすべてのクリエイティブ（資料・サイト・ LP ・バナー・ SNS）で維持する。
トンマナ＝雰囲気・印象の調子（トーン）＋それを形づくる形式・ルール（マナー）。守り続けることがブランドの認知・信頼・資産になる。

## 運用ルール（最重要・厳守）

1. **初回は必ず確認する**: あるブランドの資料・サイト・ LP ・バナー等を初めて作るときは、勝手にトンマナを決めない。テイスト候補と配色案を提示し、AskUserQuestion 等で ohcha さんと確認しながら決定する
2. **決定後はブランド内で固定**: 一度決めたら同ブランドのすべての媒体で共通。媒体でレイアウトは変えても、トーンは変えない
3. **トンマナシートに保存**: 決定内容は必ずプロジェクト内にトンマナシート（後述テンプレート）として書き出し、次回以降はまずシートを探して従う。シートがあれば再確認は不要
4. **複数ブランド案件は要注意**: 1 プロジェクトに複数ブランドが混在することが多い。どのブランドのトンマナで作るべきか少しでも迷ったら、作る前に確認する
5. **TAM/TAMCHANG 例外**: TAM を紹介する資料、TAM ・ TAMCHANG 名義の資料・ LP のみ、従来のスライドトンマナ（`tam-proposal-deck` スキル/テーマ）を適用する。このスキルでの新規決定は行わない

### 初回確認の進め方

1. ヒアリング（わかる範囲で埋めてから聞く）: ターゲット（年齢・性別・ニーズ）／業種・商材／ブランドコンセプト・強み／戦略上の狙い（高級化・親しみ・信頼など）
   - **「ターゲットにどんな感情・状態を提供したいか」が不明なら必ず追加確認する**（例: コワーキングなら「ただ静かに集中か、クリエイティブな刺激か」で配色が真逆になる）
2. キーワードを 10 個程度出し、TOP3 に絞る（性格・見た目・商品価値・モチーフの 4 切り口から）
3. 必要に応じて**ムードボード**を作る: キーワードに合う参考画像・配色の要件を言語化して共有する（gemini-image / gpt-image でイメージ生成して見せるのも有効。言葉で伝わりにくい世界観のすり合わせに使う）
4. 下記テイスト分類から候補 2〜3 案を選び、それぞれ配色案（BASE/KEY/SUB/FONT の HEX 付き）と根拠を添えて提示
5. ohcha さんの選択・調整を受けて決定 → トンマナシート化

## 設計フレームワーク

### トンマナ 5 要素（擬人化モデル）

| 要素 | 例え | 決めること |
|---|---|---|
| ロゴ | 顔 | 使用ルール（最小サイズ・余白・禁止例） |
| 配色 | 服装 | BASE/KEY/SUB/FONT と比率・濃度段階 |
| 文字と言葉 | 声色と話し方 | フォント（見出し/本文/ウエイト）＋トーンオブボイス（文体・言葉遣い） |
| 写真 | 空気感 | 明るさ・色味・構図の方向性（言葉＋参考トーンで共有） |
| 装飾 | アクセサリー | あしらいのタッチ・線の太さ・量（主役を邪魔しない最小限） |

### テイスト 7 分類 × 具体ルール（決定の中核）

| テイスト | 配色の方向 | フォント | あしらい・写真 | 適合業種例 |
|---|---|---|---|---|
| クリーン＆ビジネス | 白ベース＋青・緑系。爽やかなグラデも可 | ゴシック体中心 | 余白で区切るすっきり配置。イラストで親近感 | BtoB ・士業・医療・採用・ EC |
| キュート＆フェミニン | 淡ピンク×シャンパンゴールド等。甘すぎ防止にくすみ色併用 | 丸みのある書体＋筆記体 | リボン・線画モチーフ。装飾量は抑えめ。やわらかい光の写真 | コスメ・ブライダル・フラワー・ペット |
| クール＆スタイリッシュ | 黒・グレー・強い青＋高彩度アクセント。モノクロ写真 | シンプルなサンセリフ（太めも可） | 大小差・斜め構図・余白でメリハリ。写真は暗め・青み | IT ・ファッション・ジム・男性向け |
| ラグジュアリー＆クラシカル | 深緑×ゴールド、深紅、くすみ色。低明度 | セリフ体・明朝体・筆記体 | 装飾フレーム・和柄・質感素材。重厚な陰影の写真 | ホテル・ジュエリー・高級菓子・舞台 |
| ポップ＆アーティスティック | 補色に近い大胆配色・鮮やか多色 | 個性的なデザインフォント・大胆タイポ | 手描き風・ 3D 素材を自由に | イベント・キッズ・アート系 |
| ナチュラル＆オーガニック | 黄土色×クリーム、深緑×生成り。アースカラー低彩度 | 丸みのあるやさしい書体・にじみ明朝 | 手描きイラスト・紙テクスチャ・余白多め。自然光の写真 | 食品・カフェ・子育て・インテリア |
| SNS 特化の運用 | ブランドカラー濃淡込み 3 色以内 | テンプレで文字位置・フォント固定 | テンプレ反復＋写真だけ変化。フィルターで質感統一 | どの業種でも SNS 運用時 |

### Web 配色の 4 色区分（配色案は必ずこの形式で提示）

- **BASE**: 最大面積の背景色。白か淡いトーンが基本（暗色は高級/モダン狙いを明確にしたときのみ）
- **KEY**: ブランドを象徴する主要色。ロゴ・ボタン・リンクに一貫使用
- **SUB**: KEY の補完。統一感なら類似色相、CV 強調なら補色
- **FONT**: 純黒 #000000 は避け、#111111〜#333333 程度に明度を上げる

## 表現の決定則（if-then ルール）

制作中の迷いをなくすための条件分岐。テイスト決定後もこのルールで細部を判断する。

- **色が足りない・単調だと感じたら** → 新しい色相を追加せず、設定済みブランドカラーの**濃淡（透明度）**で対応する
- **「親しみやすさ」を出すなら** → 丸ゴシック・カジュアルな書体＋「です・ます調」で話しかける言葉遣い
- **「高級感・信頼感」を出すなら** → 明朝体・セリフ体＋彩度を落とした色（くすみ・暗め）＋言い切り・丁寧な言葉遣い
- **マルチカラー（多色）にしたいと言われたら** → 無計画に増やさず、メインを 1 色固定し、**全色の濃度（トーン）を同じ % に統一**する制約を付ける（キッズ系等でも統一感が保てる）
- **装飾・イラストを入れるなら** → 主役の引き立て役に徹し、線の太さ（例: 0.5mm）・角丸（例: 2mm）まで数値で統一する
- **写真を複数枚並べるなら** → 明るさ・色味・彩度・コントラストを同一のレタッチ基準（フィルター）で統一してから使う

## 数値基準

- 配色比率の目安: **ベース 70 : メイン 25 : アクセント 5**
- 色数: 3〜4 色構成が基本。増やすときは彩度・トーンを揃える。上限目安 5 色
- 濃淡は数値で管理: 100% / 60% / 30% など段階を固定（感覚で薄めない）
- コントラスト（WCAG）: 通常テキスト **4.5:1 以上**、大きい文字 3:1 以上。色だけに頼った識別をしない
- フォントウエイト: Thin/Light は本文に使わない。本文 Regular/Medium、見出し Bold
- 文字サイズ目安: Web 本文 14〜16px、高齢者ターゲットは 12pt 以上に拡大
- 行間: 本文 0.5〜1 文字分 ／ 字間: 文字幅の 5〜10% 程度（見出しは個別調整）
- ロゴ: 保護余白（ロゴ高さの 0.2〜0.5 倍目安）と最小サイズ（例: 印刷 15mm／デジタル 40px、「文字が読める」を基準に実物で確認）を決め、変形・色変更・影付け等の禁止 8 パターンを守る

## アンチパターン

- 媒体ごとに配色・フォント・写真トーンがバラバラで「同じブランドか？」と思わせる
- 色数を増やしすぎて印象がぼやける／担当ごとに色選びがブレる
- 色相もバラバラ・彩度もバラバラの寄せ集め配色
- 複雑すぎるルールを作って形骸化する（ルールはシンプルに、使いながら見直す）
- 写真 1 枚ずつは綺麗でも、並べるとトーンが揃っていない
- 装飾・イラストのタッチや線の太さが混在する／装飾が主役より目立つ
- 「画面が寂しい」という理由で設定外の色を無計画に足す（濃淡で対応すべき）
- 「です・ます調」と「だ・である調」の混在、フォントの種類過多（ブランドの人格がブレる）
- ロゴの周囲に余白がなく、他の要素と一体化して見える
- ターゲット不在でテイストを選ぶ（「自分の好み」で決めない）

## トンマナシート（保存テンプレート）

決定後、プロジェクト内の適切な場所（例: `docs/tonmana/{ブランド名}.md`、既存のブランドフォルダがあればそこ）に保存する。

```markdown
# {ブランド名} トンマナシート
決定日: YYYY-MM-DD ／ 確認者: ohcha

## ブランド概要
- コンセプト / ターゲット / 提供したい感情・状態 / キーワード TOP3

## テイスト
- 分類: （7 分類のどれか＋補足）

## ロゴ
- アイソレーション（余白）: 例 高さの 0.25 倍 ／ 最小サイズ: 例 印刷 15mm・デジタル 40px
- 使用禁止例: 変形・指定外の色変更・縁取り・ドロップシャドウ等

## 配色
| 役割 | 色 | HEX | 用途 |
| BASE / KEY / SUB / FONT | ... | ... | ... |
- 比率: 70/25/5 ／ 濃度段階: 100/60/30

## フォント・テキスト設定
- 見出し: ／ 本文: ／ ウエイト運用:
- 行間: 例 0.75 文字分 ／ 字間: 例 5〜10%

## トーンオブボイス
- 文体（です・ます等）／ 一人称・呼びかけ ／ 使う言葉・使わない言葉

## 写真・装飾
- 写真のレタッチ基準: 例 自然光で明るめ・彩度やや低め・黄みの温かい色味に統一
- あしらいのタッチ・量: 例 線 0.5mm ・角丸 2mm に統一、主役を引き立てる最小限

## 適用範囲・例外
- 対象媒体: ／ 例外事項:
```

## 品質ゲート（トンマナ決定の合格基準)

1. ブランドを「一人の人」として見たとき、ロゴ・色・フォント・装飾・写真の 5 要素から受ける印象に矛盾がないか
2. 配色比率（70:25:5）・ロゴ余白（0.2〜0.5A）・行間字間など、**迷わない定量ルール**が設定されているか
3. トーンオブボイス（言葉遣い）が言語化され、コピーライティング時の指針になるか
4. ルールがシンプルで、非デザイナーを含むチームでも運用できるか

## 他スキルとの連携

- 決定したトンマナは **ohcha-slide-design**（資料）・**ohcha-web-design**（サイト）・**ohcha-lp-design**（LP）・**ohcha-instagram**（SNS）が参照する下位ルールになる
- 文体・言葉遣い（トーンオブボイス）の実装は **ohcha-writing** のルールと併用する
- TAM/TAMCHANG 名義は **tam-proposal-deck** を使用（本スキルの決定対象外）
