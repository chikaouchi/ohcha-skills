# ohcha-digital-marketing-teiseki

> このスキルの用途:
> デジタルマーケティングを「再現性の高い定石」で捉え、施策の設計・監査・優先順位づけを行うための思考レンズスキル。
> 自社（ohcha）およびクライアントの Web ・ EC ・ LP ・ SNS ・広告・ CRM 施策を、
> 「顧客の 3 フェーズ（日常生活・初回購入・継続購入）」と
> 「デジタル＝コストカットとストックの手段」「Web は行動を楽にする場」という原則で整理し、
> 成果の出ない自己満足施策（無駄な仕事）を捨て、80 点までを確実に取る打ち手を導く。
> 以下のリクエストで使用する：
> 「デジタルマーケの方向性を整理して」「この施策は成果が出るか診断して」
> 「サイト改善の優先順位をつけて」「なぜ売れないのか構造で見て」
> 「顧客の購買フローを設計して」「施策が自己満足になっていないかチェックして」
> 「クライアントのデジタル戦略の骨子を作って」など、デジタルマーケ全般の設計・監査・壁打ち。
> 参考文献：垣内勇威『デジタルマーケティングの定石』（日本実業出版社, 2020）の思想を実務向けに再構成。

# デジタルマーケティングの定石スキル

## このスキルの役割

デジタルマーケティングの施策を「センス・トレンド・思いつき」ではなく、**再現性の高い定石**で判断するためのレンズ。
新しい施策を足す前に「そもそも成果が出る構造になっているか」を問い、無駄な仕事を捨てて、確実に効く打ち手に資源を寄せる。

このスキルは施策アイデアを大量に出す発想ツールではない。
むしろ**やらないことを決め、当たり前を高い精度でやり切る**ための判断基準である。

適用対象：自社（ohcha）の集客・受注、およびクライアントの Web／EC／LP／SNS／広告／CRM 施策全般。

---

## 大原則（すべての判断の土台）

### 原則 1：デジタルは魔法ではなく「コストカット」と「ストック」の手段
デジタルの真価は、既存のマーケティング機能（営業トーク・カタログ・アンケート）を安く無限に代替することと、
Web ページ・リスト・コンテンツが**半永久的に働き続けるストック**になること。
「デジタルで魔法のように売上が増える」前提の提案はしない。コストカット・ストック構築への貢献で施策を評価する。

### 原則 2：Web は「態度変容の場」ではなく「行動を楽にする場」——3 秒の壁
ユーザーとの接触は一瞬で、**3 秒を超える営業トーク（ポエム・企業理念・長い説得）は読まれずに無視される**。
Web の役割は、すでに関心のある人の期待に即座に応え、面倒をなくして行動させること。

- 施策を見たら問う：「これは**買う気のある人を楽にしているか**」「それとも**関心のない人を説得しようとしているか**」

### 原則 3：まず定石で 80 点。100 点狙いの背伸びをしない
成果の 8 割は再現性の高い当たり前（定石）で取れる。最新トレンド・才能依存のアイデアは 80 点に達してから。

### 原則 4：顧客主導。カスタマージャーニーを企業が操作しない
企業が意図した順番（トップ→機能→事例→料金→申込）で遷移させようとしない。
ユーザーは自分の都合で動く。**実際の行動**から逆算して、どこから来てもゴールに直行できる設計にする。

### 原則 5：仮説は定性から立て、データは「補強」に使う
数字を眺めても「なぜ」はわからない。データを AI に丸投げして「宝物」を探すのは時間の浪費。

- 仮説の源泉は**ユーザー行動観察**。実施ルール：①既存顧客など**5 名で十分** ②**タスクを与えない**（普段通りに使ってもらう）③**操作中に話しかけない**（自然な行動が壊れる）
- アンケートの必須設問：「初めて知ったのはいつ・どこで？」「購入の参考にした情報は？」（購買フローの事実を掴む）
- アクセス解析データは、立てた仮説の裏づけと優先順位づけに添える

## 中核フレーム：顧客の 3 フェーズ

顧客を「日常生活 → 初回購入 → 継続購入」の 3 フェーズに分け、**フェーズごとに施策の役割と定石を変える**。
提案する施策がどのフェーズ向けかを必ず明示する。

| フェーズ | 顧客の状態 | 定石 | やってはいけないこと |
|---|---|---|---|
| ① 日常生活 | ニーズがまだ発生していない | **純粋想起の獲得**＋**リスト化（ストック）**でニーズ発生を最速検知 | この段階で説得・刈り取りを狙う |
| ② 初回購入 | ニーズが発生し、情報収集・比較検討中 | **ゴール直行**：寄り道させず最短で CV へ。**ゴールのハードルを下げる** | 読ませる・遷移順を強制する・高すぎるゴール |
| ③ 継続購入 | すでに購入・契約した既存顧客 | **LTV 最大化**：定期課金化・ルーティン化で「買う」という行動自体を減らす | 放置する・毎回新規客と同じ扱いをする |

### フェーズ別の定石

**① 日常生活フェーズ**
- ゴールは「今すぐ買わせる」ではなく「思い出してもらえる状態」と「ニーズ発生を検知できる接点」をつくること
- メルマガ・ LINE 登録などの**見込み顧客リスト（ストック）**を貯め、行動トリガー（特定リンクのクリック等）でニーズ発生を検知して即アプローチする
- ノウハウ資料（ホワイトペーパー）等の交換価値でリストを獲得するのが定石

**② 初回購入フェーズ（ゴール直行）**
- 比較検討中のユーザーを**迷わせず・読ませず・最短でゴールに直行**させる。全ページからゴールへ直接リンク、ファーストビューでフォーム露出も有効
- **CV が低いときはデザインより先に「ゴールのハードル」を疑う**：いきなり有料契約・来店予約を求めていないか。資料 DL ・無料トライアル・カタログ請求などハードルの低いゴールに変更し、リードの量を確保 → インサイドセールスで引き上げる
- 高単価・嗜好性の高い商材ほどユーザーは直接営業（来店・電話）を避ける。自分で比較検討できる材料を出す

**③ 継続購入フェーズ（LTV 最大化）**
- 「毎回選んで買う」という顧客の負担を消す：**定期課金（サブスク）への移行**、または高頻度接触の**ルーティン**（定期配信・定期便）を設計する
- 既存顧客は最も費用対効果の高い資産。新規獲得の前に、この受け皿があるかを確認する

## 成果地点（KGI／KPI）の設計

- フェーズと KPI を対応させる：
  - 日常生活 → リスト獲得数・想起（指名検索）・ニーズ検知数
  - 初回購入 → ゴール到達率・ CVR ・リード獲得数（ハードルを下げた中間ゴール含む）
  - 継続購入 → LTV ・定期移行率・解約率・リピート率
- SNS フォロワー数・ PV など、**成果と因果がつながらない数字を主 KPI にしない**

## チャネルの役割整理（点を面にする）

| チャネル | 主な役割 | 定石／注意 |
|---|---|---|
| SEO | 集客（初回購入の入口＋日常生活の接点） | 検索回数の少ない KW や 10 位以下の順位変動は無視。上位 3 記事を分析してユーザーの役に立つ記事を書く正攻法のみ |
| Web 広告 | 集客（フェーズを狙い分ける） | 刈り取り広告と認知広告を混同しない |
| SNS | 認知・想起・関係維持 | 即 CV 媒体ではない。日常生活フェーズの接点として設計 |
| メール／LINE（CRM） | ストック＝リスト化とニーズ検知、継続顧客のルーティン | 定石の中で費用対効果が最も高い部類。日常生活と継続購入の両方で主役 |
| Web サイト／LP | 初回購入の受け皿（ゴール直行） | フェーズで役割を分ける。全部を 1 ページに詰めない |

## アンチパターン集（無駄な仕事・自己満足施策）

施策提案・現状監査の際、以下に当てはまるものは**赤信号**。工数を止めて疑う。

- [ ] 買う気のない層を Web で説得しようとしている／3 秒超の営業トークを読ませようとしている（原則 2 違反）
- [ ] 「ブランディング」「デザイン刷新」を主目的にしている（→ 棄却し、商品・テキスト・写真など**コンテンツの中身**の改善に目的を変更する）
- [ ] 「全ページのデザイン統一」「ポエムのようなキャッチコピー」に工数をかけている
- [ ] カスタマージャーニーを企業の意図した順番で歩かせようとしている（原則 4 違反）
- [ ] データを AI ・分析ツールに丸投げして仮説なしで「宝物」を探している（→ 先に人間の仮説を要求する）
- [ ] トラフィックの少ないページの A/B テスト、ビジネスに影響しないロングテール KW の順位変動に一喜一憂している
- [ ] アトリビューション分析を精緻化すること自体が目的化している
- [ ] サイトリニューアルの「変わった感」を成果と錯覚している
- [ ] 行動観察でユーザーにタスクを与えたり、操作中に話しかけたりしている（自然な行動が壊れる）
- [ ] 全フェーズ・全ユーザーに同じメッセージ／同じ導線を出している

## 実務での使い方（運用フロー）

### A. 現状監査モード（「この施策/サイトは成果が出るか診断して」）

1. ビジネスモデル（BtoB/BtoC ・単価・ LTV を左右する要素）を把握する
2. 各施策が**どのフェーズ（日常生活/初回購入/継続購入）を狙っているか**を仕分ける
3. フェーズと施策の役割がズレていないか照合する
4. アンチパターン集でチェックし、**即時停止すべき無駄な仕事**をリスト化する
5. 「80 点を取りこぼしている当たり前」を、凝った新施策より優先して提示する

### B. 設計モード（「デジタル戦略の骨子を作って」）

1. 顧客の 3 フェーズごとに、現状の受け皿と穴を整理する
2. フェーズごとに定石施策・成果地点・ KPI を割り当てる（下記テンプレート）
3. 仮説はユーザー行動観察・アンケートを起点に、データで裏づける前提で組む
4. TAM 提案に載せる場合は **PGST** に変換する（P：顧客主導のどの課題を解くか／G：フェーズ別 KPI／S：フェーズ設計と役割分担／T：定石ベースの具体施策）

```markdown
# デジタルマーケティング施策骨子
## 前提: ビジネスモデル [BtoB/BtoC] ／ 単価・LTV ／ ターゲットの現状フェーズ
## 日常生活フェーズ: リスト獲得手段 ／ ニーズ検知トリガー
## 初回購入フェーズ: ゴール地点（ハードル調整済みか）／ ゴール直行導線
## 継続購入フェーズ: 定期課金・ルーティンの設計
## 即時停止する無駄な仕事: [...]
## ユーザー理解調査: 行動観察（5 名・タスクなし）／ アンケート必須設問
```

### C. 壁打ちモード（「なぜ売れないのか構造で見て」）

- 症状をフェーズの問題に分解する：
  - そもそも来ていない → 日常生活フェーズ（想起・リスト・接点）の問題
  - 来るが CV しない → 初回購入フェーズの問題。**まずゴールのハードル、次に直行導線、最後にデザイン**の順で疑う
  - 一度買うが続かない → 継続購入フェーズ（定期化・ルーティン）の問題
- 該当フェーズの定石とアンチパターンに絞って原因仮説を出し、行動観察で裏づける

## 品質ゲート（施策案の合格基準）

1. 各施策が 3 フェーズ（日常生活/初回購入/継続購入）のどれ向けか明確に定義されているか
2. 初回購入フェーズの施策が「ゴール直行」構造で、ゴールのハードルは適切に下げられているか
3. 目的が「デザイン刷新」「ブランディング」等の自己満足でなく、コストカットまたはストック構築に直結しているか
4. 行動理由を知るための定性調査（行動観察 5 名・アンケート）がプロセスに組み込まれているか

## 注意

- このスキルは書籍の思想を実務用に再構成したもの。書籍本文の転載は含まない
- 「定石」は再現性の土台であり思考停止のテンプレではない。案件ごとに顧客像とフェーズを描き直すこと
- 2020 年時点の枠組みが土台。生成 AI 検索（AIO/LLMO）等の最新トレンドは別途上書きする前提で使う（＝80 点の土台＋トレンドの上積み）
